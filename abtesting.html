A/B Testing and Eye Tracking

  1. Home

The goal of this assignment is to understand the value of quantitative and qualitative user tests for evaluating different UI designs.

Hypotheses

Based on our two different homepage designs of a fictional bakery, my group and I came up with three sets of hypotheses:

    Task Success Rate

    Null Hypothesis: The success rate on Version A will be less than the success rate on Version B.

    Alternative Hypothesis: The success rate on Version A is going to be equal to version B because both designs provide add to cart and checkout functionality

    Time for Completion

    Null Hypothesis: The time until completion on Version A will be equal to or greater than the time until completion on Version B.

    Alternative Hypothesis: The time until completion on Version A is going to be less than that of Version B, because Version A has the cart directly on the menu page

    Time to first item into cart

    Null Hypothesis: The success rate on Version B will be equal to or less than the success rate on Version A.

    Alternative Hypothesis: The success rate on Version B is going to be greater than that of Version A, because Version B has red “Add to cart” buttons, which will draw the users attention more quickly than A’s purple buttons

Key Observations

Based on our results, we reject our time for completion null hypothesis. The data file for version A was longer than the data file for version B, which tells us that the user for version A took longer than the user for version B. The user was able to add a specific food item and checkout more quickly on version B than on version A. This was due to the fact that version A had items already in the cart, which may have confused the user of version A, whereas version B had a more straightforward interface. In the replay of the eye tracker, you could see the version A user look up and down the page, implying confusion. Thus, we believe that version B would be a better interface for the bakery.

It was interesting to see how even subtle design changes can have a large effect on user response to simple tasks. Being able to utilize eye tracking data and programs to visualize first-hand how users interact with design choices was eye-opening and I was able to learn a lot from this experience.
